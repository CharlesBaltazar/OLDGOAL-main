# OLDGOAL

O OLDGOAL é um sistema web voltado para o acompanhamento nutricional e controle de refeições, desenvolvido utilizando HTML, CSS, JavaScript e MySQL, com integração de servidor Node.js e Express. Seu objetivo principal é auxiliar os usuários no monitoramento de suas dietas, permitindo o registro de refeições, cálculo de calorias consumidas e acompanhamento do histórico alimentar de maneira simples e interativa. A aplicação oferece também um painel administrativo para gerenciar contas de usuários e manter o controle das informações inseridas no banco de dados.

O sistema foi estruturado para oferecer uma navegação intuitiva. Na página inicial, o usuário tem acesso às principais funções do sistema, como visualizar seu perfil, consultar nutricionistas, acessar receitas, configurar lembretes e checar a próxima refeição agendada. Além disso, a tela mostra o total de calorias consumidas no dia, com atualização automática a partir dos dados registrados nas refeições concluídas. Usuários com permissão de administrador podem acessar uma área adicional onde são listados todos os usuários cadastrados, com possibilidade de edição e exclusão de dados.

O banco de dados utilizado é o MySQL, estruturado em três tabelas principais: usuario, refeicao e refeicao_usuario. A tabela de usuários armazena informações básicas como nome, gênero, data de nascimento, e-mail, senha, biografia e tipo de conta (usuário ou administrador). A tabela de refeições guarda os dados sobre cada refeição cadastrada, incluindo nome, horário, ícone e um campo JSON contendo os alimentos e suas respectivas calorias. Por fim, a tabela refeicao_usuario relaciona o usuário às suas refeições diárias e marca se elas foram concluídas, permitindo que o sistema calcule o total de calorias ingeridas e exiba relatórios diários e históricos.

O backend foi desenvolvido em Node.js, com o framework Express, responsável por lidar com as rotas de cadastro, login, manipulação de refeições e histórico. As rotas incluem funcionalidades como criação e exclusão de refeições, atualização de alimentos e marcação de refeições concluídas. O servidor se comunica com o banco de dados para realizar operações CRUD e envia as respostas no formato JSON, sendo consumidas pelo frontend através da API Fetch. O servidor opera na porta 3022 e utiliza o middleware CORS para permitir conexões locais.

No frontend, as interações são realizadas com JavaScript puro. O sistema utiliza o localStorage para armazenar temporariamente os dados do usuário logado, garantindo persistência durante a navegação. O processo de autenticação é simples: ao se cadastrar ou realizar login, o usuário é direcionado automaticamente para a página principal, onde seus dados e refeições são carregados. Caso não esteja autenticado, o sistema impede o acesso às seções restritas e redireciona para a página de login.

O funcionamento geral do sistema segue uma lógica clara. O usuário cria uma conta informando nome, gênero, data de nascimento, e-mail e senha. Após o login, ele pode montar suas refeições adicionando alimentos com nome, quantidade e valor calórico. Cada refeição é associada a um horário e pode ser marcada como concluída. O sistema, então, soma automaticamente as calorias das refeições concluídas e exibe o total consumido no dia. Há também uma página de histórico, onde o usuário pode consultar refeições anteriores, com detalhes como data, calorias e alimentos consumidos. Já na página de perfil, é possível editar informações pessoais, alterar senha ou excluir a conta.

Para os administradores, há uma interface dedicada ao controle de usuários, onde é possível visualizar todos os cadastros, alterar dados e remover contas. Essa funcionalidade é protegida por uma verificação de permissão no servidor, garantindo que apenas administradores tenham acesso.

O manual de uso do sistema é bastante simples. Primeiro, o usuário deve iniciar o servidor executando o arquivo server.js com Node.js, garantindo que o banco de dados oldgoal esteja criado e configurado conforme o arquivo db_oldgoal.sql. Em seguida, basta abrir o arquivo index.html em um navegador. O sistema exibirá as opções de login e cadastro. Após o login, o usuário pode navegar entre as páginas para montar suas refeições, consultar o histórico e editar seu perfil. Administradores terão acesso adicional à página de gerenciamento de usuários.

Em suma, o OLDGOAL é uma aplicação funcional e intuitiva, que integra conceitos de nutrição e tecnologia para ajudar usuários a acompanharem sua alimentação de forma prática, oferecendo recursos de controle, histórico e personalização com um design leve e objetivo.
